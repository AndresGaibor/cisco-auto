QUEUE ARCHITECTURE (CANONICAL)
================================================================================

{DEV_DIR}/
├── commands/          ← Pending queue
├── in-flight/         ← Claimed (atomic rename)
├── results/           ← Source of truth (atomic write)
├── dead-letter/       ← Irrecoverable
├── lease.json         ← Lease holder info
└── journal/           ← Event logs + pending state

Flow:
  1. File Creation → commands/{seq}-{type}.json
  2. Atomic Claim → mv to in-flight/{seq}-{type}.json
  3. Execute → Handler runs
  4. Publish → results/{cmdId}.json (atomic)
  5. Cleanup → rm in-flight/{seq}-{type}.json
