# Build System

> Documented: 2026-04-14

## Pipeline Overview

```
TypeScript source files
       │
       ▼
┌─────────────────────────┐
│  AST Collection         │  Gather files per manifest (main-manifest / runtime-manifest)
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  AST Transform          │  TypeScript → ES5, strip module syntax, const→var, minify
│  (ast-transform.ts)     │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  PT-Safe Validation     │  Regex scan for forbidden patterns
│  (validate-pt-safe.ts)   │  Fail build if: import/export, const/let, arrow fn, class, etc.
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  Assembly                │  Wrap in IIFE, inject globals (ipc, fm, dprint, DEV_DIR)
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  Output                  │  Write to dist-qtscript/
└─────────────────────────┘
```

## Artifacts

### main.js (~45 KB)

**Generated by:** `renderMainV2()` from `src/pt/kernel/`

**Source manifest:** `src/build/main-manifest.ts` — 11 kernel files + 4 terminal files

**Contains:**
- `createKernel(config)` factory
- `main()` / `cleanUp()` entry points
- `_ptLoadModule(name)` — loads catalog.js and runtime.js at startup
- Command queue polling
- Runtime loader (hot-reload)
- Lease manager
- Heartbeat writer
- Job executor

### runtime.js (~15 KB)

**Generated by:** `renderRuntimeV2Sync()` from `src/handlers/`, `src/runtime/`, `src/core/`, `src/pt/`

**Source manifest:** `src/build/runtime-manifest.ts`

**Contains:**
- `runtimeDispatcher(payload, api)` — handler router
- `HANDLER_MAP` — type → handler function mapping
- All handler implementations (device, link, vlan, dhcp, host, canvas, module, inspect)
- Payload validation
- IOS parsers

**Excludes:** `catalog` (goes to catalog.js), ios-engine.ts (deprecated), ios-session.ts (deprecated)

### catalog.js (~2.5 KB)

**Generated by:** `renderCatalog()` from `src/pt-api/pt-constants.ts`

**Contains:**
- `PT_DEVICE_TYPE_MAP` — device type IDs
- `PT_CABLE_TYPE_MAP` — cable type IDs
- `PT_PORT_MAP` — port types by model
- `PT_MODULE_CATALOG` — supported modules per router model

## PT-Safe Validation Rules

Build **FAILS** if generated code contains any of:

| Pattern | Forbidden Because |
|---|---|
| `import` / `export` | QTScript doesn't support ES modules |
| `const` / `let` | QTScript only supports `var` |
| Arrow functions (`=>`) | Not ES5-compliant in QTScript |
| `class` declarations | Use prototype-based functions instead |
| `async` / `await` | Not supported in QTScript |
| `?.` optional chaining | Not ES5 |
| `` `${...}` `` template literals | Not ES5 |
| `...spread` in expressions | Not ES5 |
| `globalThis` | Not defined in PT's QTScript engine |
| `console.*` | Use `dprint` instead (PT's logging API) |
| `require()` | Use static imports or inline |
| `process` | Not available in PT sandbox |
| `Buffer` | Not available in PT sandbox |

## Validation File: `validate-pt-safe.ts`

Regex-based scanner. Runs on every artifact before writing output.

```typescript
import { validatePtSafe, formatValidationResult } from "./src/build/validate-pt-safe";
const result = validatePtSafe(code);
if (!result.valid) {
  console.error(formatValidationResult(result));
  process.exit(1);
}
```

Also detects in template strings (must use `"string" + var` instead of `` `string ${var}` ``) and spread on identifiers (`...[x]`).

**Comment handling:** Lines starting with `//` are ignored (validators skip them).

## Commands

```bash
bun run generate        # Generate all artifacts → dist-qtscript/
bun run validate        # Validate generated artifacts
bun run deploy          # Deploy to ~/pt-dev/
bun run pt:build        # From monorepo root: generate + validate + deploy
bun run pt:build --no-deploy  # Generate + validate only
bun run typecheck       # TypeScript type check (no emit)
bun run validate:api    # Validate PT API surface
```

## Generated Output Location

Default: `~/pt-dev/` (macOS/Linux)

Override: `export PT_DEV_DIR=/custom/path`
